# Lydia's Thrift — Mobile (React Native)

Expo / React Native app that talks to the Laravel API in [`../lydia-thrift-backend`](../lydia-thrift-backend).

## Requirements

### Development (Expo Go — optional)

- Node.js 20.19+
- npm
- [Expo Go](https://expo.dev/go) on a device

### Standalone builds (no Expo dev server)

The app can be built as a normal native app. JavaScript is bundled **into the APK/IPA** at build time — you do **not** need `expo start` or Expo Go to run it afterward.

| Tool | Purpose |
|------|---------|
| [Android Studio](https://developer.android.com/studio) | Android SDK, emulator, JDK |
| Xcode (macOS only) | iOS builds |

Set `ANDROID_HOME` to your Android SDK path and ensure `java` is on your PATH.

## Setup

```bash
cd lydia-thrift-mobile
cp .env.example .env
npm install
```

**Important for release builds:** set `EXPO_PUBLIC_API_URL` in `.env` **before** you build. That value is baked into the app at compile time.

| Device | Example `EXPO_PUBLIC_API_URL` |
|--------|-------------------------------|
| **Production VPS** | `http://185.252.235.149:30/api` |
| Android emulator (local backend) | `http://10.0.2.2:8080/api` |
| Physical phone (same Wi‑Fi as PC) | `http://192.168.1.10:8080/api` |
| HTTPS (when domain + cert ready) | `https://api.yourdomain.com/api` |

Start the backend first. API reference: [`../lydia-thrift-backend/docs/API.md`](../lydia-thrift-backend/docs/API.md).

## Option A — Standalone native build (recommended)

### 1. Generate native projects (first time, or after native config changes)

```bash
npm run prebuild
```

This creates `android/` and `ios/` folders. Re-run with `npm run prebuild:clean` if you change `app.json` plugins.

### 2. Build and install (Android, Windows)

Debug (installs on emulator/device, still no Expo server at runtime):

```powershell
npm run run:android
```

**Release APK** (optimized, for sharing / sideloading):

```powershell
npm run build:android
```

APK output (after Gradle finishes):

```
android\app\build\outputs\apk\release\app-release.apk
```

Or build only the APK without reinstalling:

```powershell
npm run prebuild
npm run build:android:apk
```

### iOS (macOS only)

```bash
npm run prebuild
npm run run:ios:release
```

Open `ios/lydiathriftmobile.xcworkspace` in Xcode to archive for the App Store.

---

## Option B — Expo Go (quick dev only)

```bash
npm start
```

Scan the QR code with Expo Go. This **does** use the Expo dev server and is only for fast iteration.

## Project layout

| Path | Purpose |
|------|---------|
| `src/api/` | HTTP client, Sanctum token storage, auth & products |
| `src/context/AuthContext.tsx` | Session state |
| `src/screens/` | Login, register, home (with product list) |
| `src/config/env.ts` | API base URL fallback when env is unset |
| `android/` | Generated by `prebuild` — Gradle project |
| `ios/` | Generated by `prebuild` — Xcode project (macOS) |

## Troubleshooting

| Issue | Fix |
|-------|-----|
| “Could not load products” on a **built** app | Set `EXPO_PUBLIC_API_URL` in `.env` to your PC’s LAN IP, then rebuild |
| Network error on Android | HTTP is allowed (`usesCleartextTraffic`); use `http://` for local dev |
| `SDK location not found` | Set `ANDROID_HOME` and install SDK Platform 34+ in Android Studio |
| Gradle / Java errors | Use JDK 17 (bundled with Android Studio) |
| Native build fails on Windows (`CreateProcess`, path too long) | Build only `arm64-v8a` (set in `android/gradle.properties`), or move the project to a short path without spaces (e.g. `C:\dev\lydia-thrift-mobile`) |
| `Access is denied` during CMake/ninja | Temporarily disable antivirus on the project folder, or rebuild from a shorter path |

Authentication uses **Laravel Sanctum** bearer tokens, matching the backend.
